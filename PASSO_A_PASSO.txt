2) Como usar na competição

Abra em um tablet/notebook para o juiz da Mesa 1:
pontuacao-mesa1.html

Abra em outro tablet/notebook para o juiz da Mesa 2:
pontuacao-mesa2.html

Abra no telão principal:
dashboard.html

Se quiser telão separado por mesa, use:
dashboard-mesa1.html
dashboard-mesa2.html

3) Regra do ranking

Round TESTE: aparece no histórico, mas não conta para ranking final.
Round 1: aparece no dashboard e conta para ranking final.
Round 2: NÃO mostra a marcação/nota no dashboard, mas conta para definir a melhor pontuação oficial.

Ranking final = maior nota entre Round 1 e Round 2, comparando os resultados das duas mesas.

4) Cronômetros

Cada mesa tem seu próprio cronômetro:
- Mesa 1 controla o timer da Mesa 1.
- Mesa 2 controla o timer da Mesa 2.

O Dashboard Geral usa o cronômetro da Mesa 1 como referência.
Os dashboards individuais usam o cronômetro da própria mesa.

5) Firestore

Cole esta regra em Firestore Database > Regras para teste:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

6) Publicação igual ao projeto Canecas

Suba todos os arquivos para o GitHub.
O site funciona como HTML/CSS/JS estático, conectado ao Firebase pelo firebase-config.js.

7) Observações

A coleção results salva todos os resultados, incluindo mesa, equipe, round e pontuação.
A coleção timers salva o cronômetro separado por mesa.


ACESSO AO ADMIN
Senha padrão do painel administrativo: FLL@2026MG
Abra admin.html, digite a senha e use os controles para cadastrar equipes, exportar CSV, zerar Mesa 1, zerar Mesa 2 ou zerar toda a competição.

ALTERAR VISUAL
No Admin, em Visual do sistema, coloque o caminho do banner e do fundo. Exemplo: assets/banner-topo.png.
Depois envie a imagem para a pasta assets no GitHub.

COMO SUBIR NO GITHUB

1. Entre no seu repositório no GitHub.
2. Apague a pasta public antiga, se ela existir.
3. Envie todos os arquivos desta pasta diretamente para a raiz do repositório.
4. O index.html precisa aparecer na primeira tela do repositório, junto com firebase.json.

PUBLICAÇÃO IGUAL AO CANECAS DEL REI

Use o Firebase Hosting conectado ao GitHub:

1. Acesse o Firebase Console.
2. Entre no projeto microregional-fll.
3. Vá em Hosting.
4. Clique em Começar ou Adicionar outro site, se necessário.
5. Escolha integração com GitHub.
6. Conecte o repositório do GitHub.
7. Escolha a branch main.
8. Quando perguntar a pasta pública, use:

.

Isso significa raiz do projeto.

9. Finalize a configuração.

Depois disso, toda vez que você enviar alteração para o GitHub, o Firebase publica automaticamente.

REGRAS DO FIRESTORE PARA TESTE

Use estas regras no começo:

rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

Depois, quando tudo estiver pronto, trocamos para regras com login.
